# Springで作る共有TODOリスト
Springを使用して「共有TODOリスト」を作成しました。

## 開発環境
* Windows 10 Pro ver.2009
* Eclipse 2023-12
* Java SE-21
* PostgreSQL 17

## 作成目的
今回のアプリ作成はSpringの学習の定着度合いを確認・復習をするために行いました。

## 工夫したところ
1. UserDetailsクラスでEntityUserクラスを参照できるようにし、ログイン中のユーザー情報を持たせやすくした。
2. 例外処理をコントローラアドバイスを使用して一括で管理できるようにした。
3. バリデーションアノテーションを独自で定義し、バリデーションを充実させた。
4. htmlタイトルをコントローラクラスで管理することで、保守性を高めた。
5. MapperXMLファイル内でsqlタグを使うことにより保守性を高めた。

## 改善点
1. 権限情報の仕様を決めておらず管理者権限のみをデータベースでデフォルト値として設定しているので、仕様を決めて権限機能を実装したい

## その他
* SecurityHelperはユーザー情報の更新をログアウトさせずにセッションにも反映させるために作成したが、更新後ログアウトさせて再認証させる方がよいと考えたため使わなくなった。
　しかし、今後変更の可能性を考慮し残すことにした。

* Entityクラスではjava.util.DateのDate型を使用する。
　理由：java.sql.DateはJDBC専用の型だから

* ModelAndViewクラス―引数として扱わない
　理由：呼び出し側がViewの選択などの責任を持たないように分離させるため
